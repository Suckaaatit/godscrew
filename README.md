GTM automation with optional webhook-based CRM reply tracking.

---

## Architecture

```
contacts.js (local static config)
    ↓
send-emails.js
    ↓ injects personalized copy into HTML template
    ↓ supports DRY_RUN + idempotency key per contact
    ↓ sends via Resend HTTP API (Idempotency-Key header)
    ↓ writes deterministic delivery.json state

Resend delivers to recipient inbox
    ↓
webhook-server.js (Svix verification optional/strict)
    ↓ processes inbound.email / email.opened / email.clicked / email.delivered
    ↓ optional Slack notify + Mistral intent classification
    ↓ HubSpot PATCH by email idProperty
```

Tier 1 depends only on verified Resend send capability.
Tier 2 depends on Resend inbound routing and webhook endpoint configuration.

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Configure .env

| Variable | Where to get it |
|---|---|
| `RESEND_API_KEY` | Resend → API Keys |
| `SENDER_EMAIL` | Confirm exact verified sender with Anakin |
| `SENDER_NAME` | Your name |
| `REPLY_TO_EMAIL` | Your personal email address |
| `HUBSPOT_ACCESS_TOKEN` | HubSpot → Settings → Integrations → Private Apps |
| `NGROK_STATIC_DOMAIN` | ngrok.com → Dashboard → Domains |
| `CAMPAIGN_ID` | Optional campaign tag for logs/state |
| `DRY_RUN` | `true` to simulate sends without API call |
| `RESEND_WEBHOOK_SIGNING_SECRET` | Resend webhook secret (`whsec_...`) |
| `ENFORCE_WEBHOOK_SIGNATURE` | `true` rejects unsigned webhooks |
| `SLACK_WEBHOOK_URL` | Optional Slack incoming webhook |
| `MISTRAL_API_KEY` | Optional lead intent classification |
| `MISTRAL_MODEL` | Optional model override (`mistral-small-latest`) |

### 3. HubSpot Private App scopes required

```
crm.objects.contacts.read
crm.objects.contacts.write
crm.schemas.contacts.read
crm.schemas.contacts.write
```

## Production signal features

- Dry-run mode: `DRY_RUN=true npm run send`
- Structured logs with ISO timestamps in send + webhook paths
- `delivery.json` persisted per recipient with send status and IDs
- `campaign-state.json` with live event feed + contact telemetry + last run summary
- Resend idempotency via `Idempotency-Key` header
- Execution summary block after each send run
- Svix webhook verification support (`RESEND_WEBHOOK_SIGNING_SECRET`)

## Failure handling

| Symptom | Cause | Fix |
|---|---|---|
| `[SEND] [FATAL] missing env vars` | `.env` not populated | Fill all values |
| `[SEND] [BLOCKED] .sent.lock exists` | Script already ran | Delete `.sent.lock` only to intentionally re-send all contacts |
| `[SEND] [ERROR] domain not verified` | SENDER_EMAIL not verified in Resend | Confirm exact sender with Anakin |
| `[SEND] [DRY_RUN]` logs only | DRY_RUN enabled | Set `DRY_RUN=false` for live send |
| `[SETUP] [ERROR] 403` | Private App missing schema scope | Create properties manually in HubSpot UI |
| `[HUBSPOT] [400] property_missing` | setup not run | Run `npm run setup` or create manually |
| `[HUBSPOT] [404] contact_not_found` | Email mismatch | Emails in contacts.js must match HubSpot exactly |
| `[WEBHOOK] [ERROR] webhook_rejected` | Svix signature invalid/missing | Set correct `RESEND_WEBHOOK_SIGNING_SECRET` and headers |
| Webhook never fires | Resend inbound routing not configured | Use Version B Loom script |
| ngrok URL changed | ngrok restarted | Use static domain — never restart ngrok during demo session |
